37/*
38 *Define the below as 1 if each LINKED_LIST is to store a custom
39 method for freeing a LINKED_LIST_NODE. This is useful when
40 deleting a node that has data that is referenced. For instance,
41 a node containing an object of type person. Everything is fixed
42 length except for a persons name. If their name was allocated
43 with a call to a malloc of sorts, then the nodes data gets erased
44 the allocated memory for the persons name will not get erased and
45 a memory leak is created. This allows the user to specify a
46 method that will be called when deleting a node with such data.
47 The user can first free the persons name, then the entire person
48 object so no memory leaks are created.
49*/
50#define USING_LINKED_LIST_DEPENDENT_FREE_METHOD 1
