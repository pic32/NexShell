Detailed Description

This is the interface to adding user commands. User commands are global commands that will appear in the help command list. There is a global LINKED_LIST that is used to hold the data for the commands. When the shell gets a valid command it will go through this list to see if a match occurs. If there is a match the execute method will be called.

Adds a user command to the user command list.

Parameters
[in]constchar* CommandName - The command name to associate with the command. This must be unique when compared to other user commands and native commands.
[in]constchar* Description - The description to associate with the command. This is displayed with the ls and help commands. This parameter is optional.
[in]constchar* HelpFile - The help message to display when help [command_name] is entered into the shell. This can be a string or the path of a file to display. This parameter is optional.
[in]SHELL_RESULT(*)(char*[], UINT32, PIPE*) ExecuteFile - The method to execute when the user enters the command into the shell. The char*[] argument is an array of arguments the user can send the command. The UINT32 is the number of arguments in the arg array. The PIPE* is the stream to write data output to.
Returns
SHELL_RESULT - The outcome of the operation.

This function is useful to add custom commands to the shell interface. Each command name must be unique. Description and HelpFile are not mandatory but strongly encouraged. It is also strongly encouraged to support the –help option too. The PIPE* which is passed to the ExecuteFile method can be the standard output stream or a file. The ExecuteFile method doesn't know or care as the interface for writing is always the same.
